# 概念分析Skill v1.0

## 核心方法论：四步搜索法

```
Step 1: 概念定义搜索
   └── multi_source_search("概念名称 技术原理 核心参数")
   └── 目的：确定概念的技术特征和关键指标

Step 2: 催化剂搜索
   └── multi_source_search("概念名称 订单 融资 政策 突破")
   └── 目的：找到触发行情的关键事件

Step 3: 标的映射搜索
   └── multi_source_search("概念名称 A股 上市公司 概念股")
   └── 目的：找到直接关联的A股标的

Step 4: 产业链验证搜索
   └── multi_source_search("概念上游 核心材料")
   └── multi_source_search("概念下游 应用场景")
   └── 目的：验证产业链完整性
```

**强制规则**：不完成Step 1-2，不得进入标的筛选！

## 概念分类判断法

| 用户Query特征 | 概念类型 | 处理策略 |
|:---|:---|:---|
| "XX电池/XX材料" + 特定技术词 | **细分概念** | 精准搜索技术关键词 |
| "分析XX行业" | 大类 | 用industry-chain-analysis Skill |
| "XX概念/XX主题" | 主题炒作 | 找直接关联标的 |
| "新技术/新突破" | 颠覆性技术 | 必须先搜新闻 |

**判断标准**：
- 如果概念包含**特定技术参数**（如"100小时储能"、"$20/kWh"）→ 细分概念
- 如果概念是**通用名词**（如"储能电池"、"半导体"）→ 大类

---

## 标的关联度评分

```
标的关联度 = 直接关联度 × 0.6 + 间接关联度 × 0.3 + 概念纯度 × 0.1
```

**直接关联度**：
  - 有该技术/产品：100分
  - 有研发/布局：70分
  - 有上下游材料：40分
  - 纯属概念：10分

**筛选标准**：
- 关联度 ≥ 70分：核心标的
- 关联度 40-70分：次要标的
- 关联度 < 40分：排除

## 错误快速识别信号

| 错误信号 | 表现 | 修正方法 |
|:---|:---|:---|
| 🚨 关键词漂移 | 搜索词与用户query不一致 | 回到用户query重新提取 |
| 🚨 标的泛化 | 推荐的是行业龙头而非概念标的 | 加入概念关键词重新搜索 |
| 🚨 催化剂缺失 | 没有近期订单/融资/政策 | 延长搜索时间范围 |
| 🚨 逻辑空洞 | "受益于行业发展" | 找到具体的技术/订单关联 |
| 🚨 信息源单一 | 只用了Tushare | 强制使用multi_source_search |
