# 业务线拆分估值分析 Skill v1.0

## 概述

基于**真实财报业务拆分**的精细化估值方法。

核心改进（vs v5.0收入-成本模型）：
- ✅ **按业务线拆分**: 从财报 `fina_mainbz` 获取各产品/业务线的收入和利润
- ✅ **多源搜索每条线**: 对每个业务分别搜索价格、出货、需求变化
- ✅ **分业务预测**: 分别估算每条业务线的营收和利润变化
- ✅ **最后汇总**: 各业务预测结果加总，得出整体预测

---

## 核心方法

### 数据流

```
┌─────────────────────────────────────────────────────────────────┐
│  Step 1: 财报拆分                                               │
│  • Tushare fina_mainbz 获取分产品/业务数据                       │
│  • 输出: 每条业务线的收入、利润、利润率                           │
├─────────────────────────────────────────────────────────────────┤
│  Step 2: 多源搜索（每条业务线分别搜）                            │
│  • P1 Exa: "{公司} {业务名} 价格 订单 出货"                      │
│  • P2 知识星球: 调研纪要                                         │
│  • P3 新浪财经: 公司公告/板块新闻                                │
├─────────────────────────────────────────────────────────────────┤
│  Step 3: 分业务预测                                             │
│  • 营收 = 基期营收 × (1+价格变化) × (1+出货变化)               │
│  • 利润 = 预测营收 × 利润率 × (1 - 费用调整)                    │
│  • 假设: 原料成本不变，管理/营销费用按比例变化                   │
├─────────────────────────────────────────────────────────────────┤
│  Step 4: 汇总                                                   │
│  • 总预测营收 = Σ各业务预测营收                                  │
│  • 总预测利润 = Σ各业务预测利润                                  │
└─────────────────────────────────────────────────────────────────┘
```

---

## 关键公式

### 单业务线营收预测
```
预测营收 = 基期营收 × (1 + 价格变化率) × (1 + 出货量变化率)
```

**价格变化率**来源:
- 搜索关键词中包含"涨价"/"提价"/"涨价函" → 正变化
- 搜索关键词中包含"降价"/"价格战" → 负变化
- 无明确信号 → 0%

**出货量变化率**来源:
- "订单增长"/"产能满产"/"供不应求" → 正变化
- "订单下滑"/"产能过剩"/"砍单" → 负变化

### 单业务线利润预测
```
预测利润 = 预测营收 × 基期利润率 × (1 - 费用率调整)

其中:
- 基期利润率 = 基期利润 / 基期营收 (来自财报)
- 费用率调整 = 管理/营销成本变化带来的利润率影响
- 原料成本假设不变 (已内含在基期利润率中)
```

### 费用率调整逻辑
```
营收增长时:
  - 规模效应 → 费用率下降 → 利润率上升
  - 调整: +1%~+3%

营收下降时:
  - 规模不经济 → 费用率上升 → 利润率下降
  - 调整: -1%~-3%

竞争激烈/扩张期:
  - 营销费用增加 → 利润率下降
  - 调整: -2%~-5%
```

## 使用方法

### 命令行

```bash
# 基础用法
./venv_runner.sh skills/segmented-business-forecast/scripts/segmented_forecaster.py 300308.SZ --name "中际旭创"

# 输出到文件
./venv_runner.sh skills/segmented-business-forecast/scripts/segmented_forecaster.py 300308.SZ --name "中际旭创" -o report.md
```

### Python API

```python
import sys
sys.path.insert(0, '/root/.openclaw/workspace/skills/segmented-business-forecast/scripts')

from segmented_forecaster import SegmentedBusinessForecaster

# 创建分析器
forecaster = SegmentedBusinessForecaster()

# 加载股票
forecaster.load_stock('300308.SZ', '中际旭创')

# 执行完整预测
result = forecaster.run_forecast()

# 生成报告
report = forecaster.generate_report()
print(report)
```

### 分步调用

```python
# Step 1: 获取业务拆分
forecaster.fetch_business_segments()
forecaster.display_segments()

# Step 2-3: 逐个业务搜索+预测（手动控制）
for segment in forecaster.segments:
    dynamics = forecaster.search_business_dynamics(segment)
    forecaster.forecast_segment(segment, dynamics)

# Step 4: 汇总
forecaster.summarize()
```

## 关键假设与限制

### 假设
1. **原料成本不变**: 各业务线的原材料成本结构保持基期水平
2. **管理/营销成本按比例变化**: 默认+5%，可根据搜索结果调整
3. **价格/出货变化基于搜索信号**: 定性判断转为定量估算

### 限制
- ⚠️ `fina_mainbz` 数据可能不完整（部分公司不披露分产品利润）
- ⚠️ 多源搜索结果为定性信号，定量转化存在主观性
- ⚠️ 未考虑业务间协同效应（如交叉销售、供应链共享）
- ⚠️ 未考虑新业务线的出现或旧业务线的退出

## 适用场景

✅ **业务多元化公司**: 如同时做光模块+汽车电子+传感器的公司
✅ **周期性行业**: 不同产品周期不同，需分别判断
✅ **并购后公司**: 需要分别评估原有业务和新增业务
✅ **转型期公司**: 老业务衰退+新业务增长，需拆分看趋势

❌ **单一业务公司**: 直接用v5.0收入-成本模型更简单
❌ **业务划分模糊**: 如"其他"占比过高（>30%）的公司

---

## 与v5.0收入-成本模型的对比

| 维度 | v5.0 收入-成本模型 | v1.0 业务线拆分 |
|:---|:---|:---|
| **核心方法** | 标杆企业对比 | 财报业务拆分 |
| **数据需求** | 需找3+标杆企业 | 只需自身财报 |
| **粒度** | 公司整体 | 分产品/业务线 |
| **适用场景** | 单一业务/行业标准化 | 多元化/周期性/转型期 |
| **复杂度** | 中 | 高 |
| **准确性** | 依赖标杆选择 | 依赖业务拆分完整度 |
